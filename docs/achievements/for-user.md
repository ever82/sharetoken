# ShareTokens User Achievements

> 用户视角下的可验收成果清单。每个 Achievement 代表用户可以体验到的价值交付。
>
> **优先级定义:**
> - **P0**: 核心路径，MVP 必须有
> - **P1**: 完整体验，MVP 后第一批
> - **P2**: 增长与差异化功能
> - **P3**: 规模化与增强功能

---

## P0 - 核心路径 (MVP)

### ACH-USER-001. Secure Digital Wallet

**优先级:** P0
**用户故事:** 作为一个用户，我想要安全地管理我的数字资产，**以便于** 我可以完全掌控自己的资金，不依赖任何中心化机构。

**验收标准:**
- [ ] 我可以创建/导入钱包（从打开网页到完成 < 60s）
- [ ] 我可以查看 STT 余额和交易历史
- [ ] 我可以安全地转账和收款
- [ ] 我可以使用 Keplr 或手机钱包管理资产
- [ ] 我可以随时导出我的私钥

**用户价值:**
- **资产主权**: 私钥自持，不托管给任何第三方
- **跨设备**: 桌面 + 移动端无缝切换
- **透明历史**: 所有交易可追溯

**关联 Spec:** CORE-003

---

### ACH-USER-002. One-Click AI Access

**优先级:** P0
**用户故事:** 作为一个没有技术背景的用户，我想要无需任何配置就能使用 AI 服务，**以便于** 我可以专注于创意和业务，而不是浪费时间在复杂的技术配置上。

**验收标准:**
- [ ] 我可以通过 GenieBot 开始对话，无需安装任何软件
  - **验证**: 在未安装任何客户端的设备上完成首次对话
  - **时间要求**: 从打开网页到发送第一条消息 < 60s
- [ ] 我可以自然语言描述需求，系统自动理解并推荐服务
  - **准确率**: 意图识别准确率 >= 85%
  - **响应时间**: 推荐结果返回 < 3s
- [ ] 我可以一键调用至少 5 种主流 AI 模型
- [ ] 我可以查看每次调用的费用明细（包含 token 数量、单价、总费用）

**用户价值:**
- **消除技术壁垒**: 无需安装 Python、配置 API Key、管理环境变量
- **零学习成本**: 像聊天一样使用 AI，自然语言即指令
- **即时可用**: 从想法到执行只需 3 步（描述 -> 确认 -> 获取结果）

**关联 Spec:** GENIE-001~002, PLUGIN-002

---

### ACH-USER-003. Fund Security Guarantee

**优先级:** P0
**用户故事:** 作为一个用户，我想要确保我的资金安全，**以便于** 我可以放心地使用平台服务，不用担心被骗或资金丢失。

**验收标准:**
- [ ] 我在服务开始前付款，资金进入托管
- [ ] 我在确认满意后资金才释放给服务者
- [ ] 如果有问题，我可以申请冻结资金
- [ ] 争议解决后，资金按裁决分配
- [ ] 我可以随时查看托管资金状态

**用户价值:**
- **交易保护**: 资金在确认交付前由智能合约托管
- **风险可控**: 不满意可申诉，资金不会自动划走
- **透明状态**: 实时查看资金处于哪个阶段

**关联 Spec:** CORE-005, TRUST-004

---

### ACH-USER-004. Transparent Service Pricing

**优先级:** P0
**用户故事:** 作为一个用户，我想要清楚地知道每项服务的费用，**以便于** 我可以合理规划消费，避免意外扣费。

**验收标准:**
- [ ] 我可以在服务市场浏览所有可用服务
- [ ] 我可以查看每个服务的定价模式（按token/按skill/打包）
- [ ] 我可以比较不同提供者的价格和评分
  - **最低显示**: 至少显示 3 个提供者
  - **评分维度**: 质量/响应速度/完成率
- [ ] 我可以在消费前看到预估费用
- [ ] 我可以查看详细的消费账单（支持导出 CSV）

**用户价值:**
- **消费前置预估**: 执行前明确告知费用，杜绝意外扣费
- **实时账单透明**: 每笔消费可追溯
- **价格比较自由**: 同一服务多个提供者报价，用户自主选择

**关联 Spec:** CORE-004, MARKET-001~004

---

### ACH-USER-005. First-Time Onboarding

**优先级:** P0
**用户故事:** 作为一个新用户，我想要快速了解平台能为我做什么，**以便于** 我可以决定是否投入时间深入学习。

**验收标准:**
- [ ] 我可以用微信/GitHub/Google 一键登录
- [ ] 钱包自动创建（无需记助记词，可后续导出）
- [ ] 我可以获得初始测试代币（水龙头）
- [ ] 首次使用有交互式引导教程（3-5 分钟）
- [ ] 我可以看到 "What can I do here" 常见用例列表

**用户价值:**
- **零门槛入门**: 3 分钟内完成注册并体验核心功能
- **风险试错**: 测试代币让新用户无风险体验
- **清晰指引**: 引导教程降低学习成本

**关联 Spec:** CORE-002, CORE-003

## P1 - 完整体验

### ACH-USER-006. Task Progress Tracking

**优先级:** P1
**用户故事:** 作为一个委托任务的用户，我想要实时了解任务进展，**以便于** 我可以及时发现偏差、调整需求，避免最后交付时才发现不符合预期。

**验收标准:**
- [ ] 我可以查看所有进行中的任务列表
  - **分页**: 支持分页，每页 >= 10 条
  - **排序**: 支持按时间/优先级/状态排序
- [ ] 我可以查看每个任务的当前状态和进度百分比
  - **更新频率**: 状态变更后 10 秒内更新
  - **进度精度**: 百分比显示，精确到 1%
- [ ] 我可以查看里程碑完成情况
- [ ] 我可以收到关键节点的通知
  - **通知渠道**: 站内消息 + 可选邮件
  - **延迟要求**: 事件触发后 30 秒内送达
- [ ] 我可以与服务者沟通（留言/补充需求）

**用户价值:**
- **实时掌控**: 随时了解任务处于哪个阶段
- **及时干预**: 发现问题可以立即调整
- **沟通顺畅**: 直接与服务者交流，减少误解

**关联 Spec:** MARKET-002, MARKET-004

---

### ACH-USER-007. Verified Identity Benefits

**优先级:** P1
**用户故事:** 作为一个实名认证用户，我想要享受更高的额度和信任，**以便于** 我可以进行更大规模的交易，并获得更多商业机会。

**验收标准:**
- [ ] 我可以通过微信/GitHub/Google 完成实名认证
- [ ] 我的认证状态显示在我的资料中（验证通过标记）
- [ ] 我可以获得更高的交易限额（具体数值见限额表）
- [ ] 我可以获得更高的提现限额
- [ ] 我可以被选为陪审员参与争议裁决

**用户价值:**
- **信任背书**: 认证标记让其他用户更信任我
- **权限提升**: 更高的交易/提现限额
- **参与治理**: 有资格参与争议裁决

**关联 Spec:** IDENTITY-001, IDENTITY-002

### ACH-USER-008. Fair Dispute Resolution

**优先级:** P1
**用户故事:** 作为一个遇到问题的用户，我想要公平地解决争议，**以便于** 我的权益得到保障，不会为不满意的服务付费。

**验收标准:**
- [ ] 我可以对任何交易发起争议
- [ ] 我可以通过 AI 调解员陈述我的观点
- [ ] 我可以提交证据（截图、聊天记录等）
- [ ] 如果 AI 调解失败，我可以选择陪审团裁决
- [ ] 我可以看到裁决结果和理由
- [ ] 我可以查看我的 MQ 分数变化

**用户价值:**
- **公正裁决**: AI + 陪审团双重机制确保公平
- **证据保全**: 可以提交各类证据支持主张
- **透明流程**: 整个争议过程可追溯

**关联 Spec:** CORE-006, TRUST-001~003

### ACH-USER-009. Service Failure & Refund

**优先级:** P1
**用户故事:** 作为一个遇到服务问题的用户，我想要知道如何获得退款，**以便于** 我的权益得到保障，不会为不满意的服务付费。

**验收标准:**
- [ ] 我可以在任务详情页发起退款申请
- [ ] 我可以选择退款原因（服务未交付/质量不达标/超时）
- [ ] 我可以查看退款处理进度
- [ ] 退款审核在 48 小时内完成
- [ ] 退款成功后 STT 在 10 分钟内返还到账

**用户价值:**
- **消费者保护**: 不满意可以申请退款
- **快速处理**: 明确的处理时限
- **透明进度**: 随时查看退款状态

**关联 Spec:** CORE-005, TRUST-004

- [ ] 我可以查看被选为陪审员的记录
- [ ] 我可以查看我的贡献统计（交易量/评价/服务次数）

**用户价值:**
- **信誉可视化**: 清晰了解自己的社区地位
- **权益透明**: 知道当前等级可以享受什么权益
- **行为激励**: 看到良好行为带来的正向反馈

**优先级:** P1
**用户故事:** 作为一个用户，我想要在一个可信的社区中参与，**以便于** 我可以放心地与陌生人交易，不用担心被骗。

- [ ] 我可以看到类似想法的参考案例

**用户价值:**
- **创意孵化**: AI 辅助将模糊想法具体化
- **成本预估**: 提前知道实现需要多少资源
- **案例参考**: 借鉴类似项目的成功经验

## P3 - 规模化

### ACH-USER-017. Multi-Agent Task Execution

**优先级:** P3
**用户故事:** 作为一个需要复杂任务的用户，我想要一个团队为我工作，**以便于** 我可以一次性解决复杂问题，而不是分别寻找多个服务。

**验收标准:**
- [ ] 我可以描述一个复杂需求（如"开发一个网站"）
- [ ] 系统自动分解为子任务
- [ ] 我可以看到多个 Agent 协作执行
- [ ] 我可以查看每个子任务的负责 Agent 和进度
- [ ] 我可以验收阶段性成果
- [ ] 最终交付完整成果

**用户价值:**
- **一站式解决**: 复杂任务不需要自己协调多个服务
- **自动分解**: 系统智能拆分任务
- **全程可视**: 看到每个 Agent 在做什么

**关联 Spec:** MARKET-004, PLUGIN-001

---

### ACH-USER-018. Cross-Platform Access

**优先级:** P3
**用户故事:** 作为一个用户，我想要在不同设备上使用 ShareTokens，**以便于** 我可以随时随地管理我的任务和资产。

**验收标准:**
- [ ] 我可以在桌面浏览器使用完整功能
- [ ] 我可以在手机浏览器使用核心功能：
  - 查看 STT 余额和交易历史
  - 发起和查看任务状态
  - 接收和查看通知
- [ ] 我可以通过 WalletConnect 连接移动钱包
- [ ] 我的账户和数据在所有设备同步
- [ ] 我可以安全地在任何设备登出

**用户价值:**
- **随时访问**: 移动端也能管理核心功能
- **无缝同步**: 所有设备数据一致
- **安全登出**: 支持远程登出其他设备

**关联 Spec:** CORE-003, REQ-001

---

### ACH-USER-019. Data Export & Account Deletion

**优先级:** P3
**用户故事:** 作为一个用户，我想要掌控我的个人数据，**以便于** 我可以迁移数据或彻底删除我的账户。

**验收标准:**
- [ ] 我可以一键导出所有个人数据（JSON/CSV 格式）
- [ ] 导出包含：交易记录、任务历史、评价、设置
- [ ] 我可以申请永久删除账户
- [ ] 删除前有 30 天冷静期，期间可撤销
- [ ] 删除后所有链上数据只保留哈希，不保留明文

**用户价值:**
- **数据主权**: 可以随时获取自己的数据
- **隐私保护**: 可以彻底删除账户和数据
- **合规保障**: 符合 GDPR 等隐私法规

**关联 Spec:** IDENTITY-001

---

**优先级:** P3

- **无障碍使用**: 用母语操作更顺畅
- **全球化**: 支持更多地区的用户
- **跨语言交流**: AI 翻译帮助理解外语服务

**关联 Spec:** GENIE-001
